# C言語ソースコード生成

## 概要

プログラム指示書（Markdown形式）を入力として、C言語ソースコードを生成するスキル。
プロンプトテンプレートを必ず使用し、変数定義部（`.h`）と関数実装部（`.c`）の2ファイルを出力する。

遵守事項・制約・禁止事項は [`.bob/rules-c-implementer/AGENTS.md`](../../rules-c-implementer/AGENTS.md) を正本とする。

## 実行手順

以下の手順を順番に実行してください。

### Step 1: セッション記憶の開始

[`MEMORY/`](../../../MEMORY/) 配下にセッションログを作成する。
ファイル名: `MEMORY/sessions/YYYY-MM-DD_session-implement-NNN.md`（当日既存の最大番号 +1）

- 既存の終了済みセッションログへ追記してはならない
- 実行中の重要事項、エラー、ユーザーフィードバックを随時記録する
- 詳細は [`.bob/rules/memory.md`](../../rules/memory.md) 参照

### Step 2: 承認済み成果物の確認

- 引数でファイルパスが指定されている場合はそれを使用
- 指定がない場合は `インプット/` 配下の最新 `.md` ファイルを検索
- 以下を確認:
  - プログラム指示書（必須）
  - 承認済みテストケース（オプション、TSV形式必須）
  - 承認済みテストデータ（オプション、入力データ `INxx.tsv` と出力データ `OUTxx.tsv` の両方が必須）
- **不適切な場合の対応**:
  - テストケースが Markdown形式のみの場合 → TSV形式への変換を促す
  - テストデータの片方が不足 → 前工程（テストデータ生成）に戻るよう案内する

入力制約の詳細は [`.bob/rules-c-implementer/AGENTS.md`](../../rules-c-implementer/AGENTS.md) 参照。

### Step 3: 入力ファイルの読み込み

**読み込み前に必ず `list_files` で存在確認してから `read_file` を実行すること。**

確認対象ディレクトリ: `インプット/`、`セルフチェックリスト/`、`プロンプトテンプレート/ソースコード/`、承認済み成果物の配置先

以下の順序で読み込む:

1. **PGM指示書**（Markdown形式）— 必須
   - `インプット/` 配下の対象 `.md` ファイル
   - ファイル内の画像リンクも確認すること
2. **プロンプトテンプレート（ソースコード生成）** — 必須
   - `プロンプトテンプレート/ソースコード/変数定義/SystemPrompt.txt` と `UserPrompt.txt`
   - `プロンプトテンプレート/ソースコード/コード生成/SystemPrompt.txt` と `UserPrompt.txt`
3. **セルフチェックリスト** — 必須
   - `セルフチェックリスト/` 配下のソースコードチェックリスト
4. **設計規約・コーディング規約** — オプション
   - `インプット/` 配下の規約類 `.md` ファイル（存在する場合）
5. **承認済みテストケース** — オプション
   - `アウトプット/` 配下の `*_テストケース.tsv`（Step 2 で確認済みのもの）
6. **承認済みテストデータ** — オプション（Step 2 で入力可と確認済みの場合）
   - 入力データ: `アウトプット/{PGM_ID}_テストデータ/IN01.tsv` など
   - 出力データ: `アウトプット/{PGM_ID}_テストデータ/OUT01.tsv` など
7. **参考資産** — 推奨
   - `参考資産/` 配下の生成例（存在しなければスキップ）

### Step 4: モード切り替え

`switch_mode` ツールを使用して `c-implementer` モードに切り替える（存在する場合）。

### Step 5: ソースコード生成

各プロンプトテンプレートの `SystemPrompt.txt` と `UserPrompt.txt` を使用して生成する。
プログラム指示書の内容に基づき、以下の順序で生成する。

ソースコードの標準構造・非自明ルールは [`.bob/rules-c-implementer/AGENTS.md`](../../rules-c-implementer/AGENTS.md) 参照。

#### Step 5-1: ファイルヘッダーの生成

- [`成果物テンプレート/テンプレート_PGM指示書.md`](../../../成果物テンプレート/テンプレート_PGM指示書.md) のヘッダー構造に従う
- プログラム指示書の概要から PROGRAM NO、NAME 等を抽出する
- `.c` ファイルと `.h` ファイルの**両方**にファイルヘッダーを付与する

#### Step 5-2: 変数定義部の生成（ヘッダーファイル）

- [`プロンプトテンプレート/ソースコード/変数定義/`](../../../プロンプトテンプレート/ソースコード/変数定義/) の `UserPrompt.txt` を使用
- **出力先: `アウトプット/{PGM_ID}.h`**
- 以下の要素を生成する:
  - インクルードガード（`#ifndef {PGM_ID}_H` / `#define {PGM_ID}_H` / `#endif`）
  - ヘッダファイルインクルード
  - マクロ定義・定数定義
  - 構造体定義
  - 関数プロトタイプ宣言
- `.c` ファイル冒頭に `#include "{PGM_ID}.h"` を追加する

#### Step 5-3: 関数実装部の生成

- [`プロンプトテンプレート/ソースコード/コード生成/`](../../../プロンプトテンプレート/ソースコード/コード生成/) の `UserPrompt.txt` を使用
- 詳細処理説明書から各関数（main関数、共通関数、サブ関数）を生成する
- **出力先: `アウトプット/{PGM_ID}.c`**
- 詳細は各ディレクトリの `README.md` 参照

### Step 6: セルフチェック実施

ユーザーレビュー前に以下を実施し、不足・誤りがあれば修正案を提示する。

1. `セルフチェックリスト/` 配下のソースコードチェックリストの観点で検証する
   - Markdownファイル内で参照されている画像リンクも必ず確認すること
2. **プロンプトテンプレートを使用して生成したかどうかを確認**
3. **関数単位でプログラム指示書とソースコードを比較し、全ての処理が一致していることを確認**

**主要チェック観点:**
- プログラム指示書の内容を過不足なく反映しているか
- 変数定義（`.h`）と処理本体（`.c`）の整合が取れているか
- 命名、インデント、戻り値、エラー処理、コメントの整合性
- インクルードガードが正しく設定されているか
- `#include "{PGM_ID}.h"` が `.c` ファイルの先頭に記述されているか

### Step 7: 出力内容

以下の成果物を `アウトプット/` へ出力する:

| ファイル | 内容 |
|:--|:--|
| `アウトプット/{PGM_ID}.h` | 変数定義部（インクルードガード、マクロ定義、構造体定義、関数プロトタイプ宣言） |
| `アウトプット/{PGM_ID}.c` | 関数実装部（`#include "{PGM_ID}.h"` を先頭行に含む） |

仮定がある場合は**仮定一覧**（曖昧な仕様への対応）も合わせて提示する。
レビュー時の確認ポイントがあれば**注意事項**として提示する。

### Step 8: ユーザー確認

以下の文言で確認を求める:

「ソースコードを承認しますか？修正が必要な箇所はありますか？」

- **承認された場合**: Step 9（セッション記憶の完了）へ進む
- **修正が必要な場合**: 修正内容を反映して Step 5 から再生成する
- **未承認のまま完了扱いにすることは禁止**

### Step 9: セッション記憶の完了

[`MEMORY/`](../../../MEMORY/) のセッションログを完成させる。
エラーや新しい学びがある場合は `MEMORY/reflections/YYYY-MM-DD_reflection.md` を生成し、
`MEMORY/sessions/index.md` と `MEMORY/reflections/index.md` を更新する。
リフレクションは `MEMORY/USAGE_GUIDE.md` のテンプレートに厳密に従うこと。

詳細は [`.bob/rules/memory.md`](../../rules/memory.md) 参照。
