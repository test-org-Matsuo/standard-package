# 指示

C言語ソースコードを読み込み、プロンプトテンプレートを利用して、プログラム指示書を生成します。

## 実行フロー

### 1. セッション記憶の開始

- 実行開始時に `MEMORY/sessions/YYYY-MM-DD_session-c-instruction-NNN.md` を新規作成する
- 当日既存ファイルの最大番号に `+1` した番号を採番する
- 既存の終了済みセッションログへ追記してはならない
- 実行中の重要事項、エラー、ユーザーフィードバックを随時記録する
- 詳細は [`.bob/rules/memory.md`](../../rules/memory.md) 参照

### 2. 入力ファイルの確認

- 引数で指定されたC言語ソースコードを読み込む
- プログラムIDを抽出（ファイル名から`{PGM_ID}.c`, `{PGM_ID}.h`, `{PGM_ID}.txt`などの形式）
- ソースコード中の主要構成を確認：
  - ヘッダファイルインクルード（`#include`）
  - マクロ定義（`#define`）
  - 構造体定義（`struct`）
  - 関数プロトタイプ宣言
  - グローバル変数宣言
  - `main`関数および各関数定義
  - ファイル入出力処理（`fopen`, `fread`, `fwrite`等）
  - データベース接続処理（該当する場合）
  - メッセージ出力、関数呼び出し、入出力編集処理

### 3. プログラム指示書生成

各プロンプトテンプレートのSystemPrompt.txt、UserPrompt.txtを使用してLLM実行する。
C言語ソースコードの内容に基づき、以下の順序で生成。

#### Step 1: 指示書全体の骨格作成
- [`成果物テンプレート/テンプレート_PGM指示書.md`](../../../成果物テンプレート/テンプレート_PGM指示書.md)の構成に従う
- プログラムID、プログラム名称、更新履歴など、ソースコードから取得できる基本情報を整理する

#### Step 2: 基本情報の生成
- [`プロンプトテンプレート/プログラム指示書/PGM概要説明書（概要）/`](../../../プロンプトテンプレート/プログラム指示書/PGM概要説明書（概要）/)
- [`プロンプトテンプレート/プログラム指示書/プログラム構造図/`](../../../プロンプトテンプレート/プログラム指示書/プログラム構造図/)
- [`プロンプトテンプレート/プログラム指示書/変数定義/`](../../../プロンプトテンプレート/プログラム指示書/変数定義/)
- 詳細は各ディレクトリのREADME.md参照

#### Step 3: 詳細情報の生成
- [`プロンプトテンプレート/プログラム指示書/サブプログラム説明書（概要）/`](../../../プロンプトテンプレート/プログラム指示書/サブプログラム説明書（概要）/)
- [`プロンプトテンプレート/プログラム指示書/サブプログラム説明書（処理仕様）/`](../../../プロンプトテンプレート/プログラム指示書/サブプログラム説明書（処理仕様）/)
- [`プロンプトテンプレート/プログラム指示書/サブプログラム説明書（エラー一覧）/`](../../../プロンプトテンプレート/プログラム指示書/サブプログラム説明書（エラー一覧）/)
- [`プロンプトテンプレート/プログラム指示書/詳細処理説明書/`](../../../プロンプトテンプレート/プログラム指示書/詳細処理説明書/)
- データベース接続がある場合は以下を使用：
  - [`プロンプトテンプレート/プログラム指示書/DB_CALL処理/`](../../../プロンプトテンプレート/プログラム指示書/DB_CALL処理/)
- 詳細は各ディレクトリのREADME.md参照

#### Step 4: 生成対象の取捨選択
- ソースコードに存在しない要素（例: データベース未使用、外部ライブラリ未使用）は空欄または対象外として扱う
- テンプレート構成は維持しつつ、該当情報がない箇所は明示的に不足情報として整理する

### 4. 統合とフォーマット

- [`成果物テンプレート/テンプレート_PGM指示書.md`](../../../成果物テンプレート/テンプレート_PGM指示書.md)の構造に従って統合
- 図表生成が必要な項目は、Markdown本文と対応するPlantUMLまたは表形式を整合させる
- 出力イメージは[`参考資産/`](../../../参考資産/)を参照

### 5. 出力

- ファイル名: `{PGM_ID}_PGM指示書.md`
- 配置先: [`アウトプット/`](../../../アウトプット/)

## 品質チェック

- 各プロンプトテンプレートのREADME.md「生成結果のチェック観点」参照
- [`セルフチェックリスト/`](../../../セルフチェックリスト/)に格納されたチェック資材がある場合はそれに従う
- セクション番号、見出し名、順序が[`成果物テンプレート/テンプレート_PGM指示書.md`](../../../成果物テンプレート/テンプレート_PGM指示書.md)と一致していることを確認する

## 実行例

```bash
/c-instruction インプット/PXYG0123.c
# → アウトプット/PXYG0123_PGM指示書.md
```
