# セッション記憶システム

このディレクトリは、エージェントがセッションを跨いで学習・改善するための記憶システムです。

## 📁 ディレクトリ構造

```
MEMORY/
├── sessions/              # ログ層
│   ├── YYYY-MM-DD_session-NNN.md
│   └── index.md
├── reflections/           # リフレクション層
│   ├── YYYY-MM-DD_reflection.md
│   └── index.md
├── knowledge/             # 記憶層
    ├── persistent.md      # 永続的な知識
    ├── patterns.md        # 成功/失敗パターン
    └── preferences.md     # ユーザー設定・好み
└── archive/               # 長期保管用
```

## 🎯 各層の役割

### 📝 ログ層 (`sessions/`)
- **目的**: セッション内の活動を時系列で記録
- **フォーマット**: Markdown（自然言語中心）
- **内容**: ユーザーとエージェントのやり取り、発生した問題、解決方法
- **更新タイミング**: セッション中にリアルタイム

### 🔍 リフレクション層 (`reflections/`)
- **目的**: セッションから重要な学びを抽出・整理
- **フォーマット**: Markdown（構造化）
- **内容**: 
  - 発生した誤り（原因・影響・対策）
  - 無駄な作業（内容・時間・改善策）
  - 成功パターン（アプローチ・効果・再現性）
  - 次回への引き継ぎ事項
- **更新タイミング**: セッション終了時

### 🧠 記憶層 (`knowledge/`)
- **目的**: 次回セッションで参照する知識を保持
- **フォーマット**: Markdown（カテゴリ別整理）
- **内容**:
  - `persistent.md`: プロジェクト固有ルール、技術制約
  - `patterns.md`: 成功/失敗パターン、試行錯誤の記録
  - `preferences.md`: ユーザーの好み、ワークフロー設定
- **更新タイミング**: リフレクション生成後、重要な学び発生時

## 🔄 記憶のライフサイクル

```
1. セッション実行
   ↓
2. ログ記録（sessions/）
   ↓
3. セッション終了
   ↓
4. リフレクション生成（reflections/）
   ↓
5. 知識抽出・統合（knowledge/）
   ↓
6. 次回セッション開始時に読み込み
```

## 📖 使用方法

### セッション開始時
1. `knowledge/persistent.md` を読み込み（必須）
2. `knowledge/preferences.md` を読み込み（必須）
3. `reflections/[最新3件].md` を読み込み（推奨）
4. `knowledge/patterns.md` を読み込み（タスクに応じて）

### セッション中
1. 重要なやり取りを `sessions/YYYY-MM-DD_session-NNN.md` に記録
2. エラー発生時は詳細を記録
3. ユーザーフィードバックを記録

### セッション終了時
1. セッションログを完成させる
2. リフレクションを生成（エラーまたは学びがある場合）
3. 知識ベースを更新

## 🎓 記録すべき重要事項

### ❌ 誤り
- 間違った実装アプローチ
- 誤解した仕様
- 不適切なファイル構造の提案
- ツール使用の誤り

### ⏱️ 無駄
- 不要なファイル読み込み
- 冗長な質問の繰り返し
- 既に試して失敗したアプローチ
- 参考資産の無駄な参照

### 💡 次回への引き継ぎ
- プロジェクト固有のルール
- ユーザーの好み・スタイル
- 成功したパターン
- 技術的な発見・制約

## 🔧 メンテナンス

### 定期的な見直し
- **週次**: パターンの有効性確認
- **月次**: 知識ベースの整理・統合

### クリーンアップ
- 3ヶ月以上前のセッションログはアーカイブする
- 使用されていないパターンは削除検討
- 矛盾する知識は統合・解決

## 📝 記録例

### セッションログの例
```markdown
## 10:05 - ファイル名の誤り発生

**問題**: 
ファイル名を`SAMPLE.cbl`として生成しようとした。

**原因**: 
PGM-IDの確認を怠り、推測でファイル名を決定した。

**ユーザーのフィードバック**:
「ファイル名が間違っています。正しくはSAMPLE01.cblです」

**エージェントの修正**:
- PGM指示書を再確認し、PGM-ID = "SAMPLE01"を確認
- 正しいファイル名`SAMPLE01.cbl`で再生成

**所要時間**: 約5分のロス
```

### リフレクションの例
```markdown
## ❌ 発生した誤り

### 1. ファイル名の誤認識
- **発生箇所**: SAMPLE.cbl → SAMPLE01.cbl
- **原因**: PGM指示書のPGM-ID読み取り不足
- **影響**: 1回のリトライ、5分のロス
- **対策**: PGM-ID確認を必須チェックリストに追加
- **次回への教訓**: ファイル名生成前に必ずPGM-IDを明示的に確認
```
