## 概要

本フレームワークは、C言語レガシーシステムの開発・保守作業において、以下の成果物をAIで生成することで、開発効率を大幅に向上させます：

- **単体テストケース**: プログラム指示書から正常系・異常系・境界値を網羅したテストケースを生成
- **単体テストデータ**: 承認済みテストケースから実行可能な入力データ・期待出力データを生成
- **C言語ソースコード**: プログラム指示書から変数定義部（`.h`）と関数実装部（`.c`）を生成
- **プログラム指示書**: 既存ソースコードからドキュメントを生成（逆引き）

各工程は **プログラム指示書 → テストケース → テストデータ → ソースコード** の順に進めることを想定しています。

## 主な機能

### 1. テストケース生成

プログラム指示書を入力として、単体テストケースをTSV形式で生成します。

**入力**: プログラム指示書（Markdown形式）  
**出力**: テストケース（TSV形式）

**生成内容**:
- 正常系テストケース
- 異常系テストケース
- 境界値テストケース

### 2. テストデータ生成

承認済みテストケースを入力として、実行可能なテストデータをTSV形式で生成します。

**入力**: 承認済みテストケース（TSV形式）、プログラム指示書  
**出力**: 入力データ（`INxx.tsv`）・期待出力データ（`OUTxx.tsv`）

### 3. ソースコード生成

プログラム指示書を入力として、C言語ソースコードを自動生成します。

**入力**: プログラム指示書（Markdown形式）  
**出力**: 変数定義部（`.h`ファイル）、関数実装部（`.c`ファイル）

**生成内容**:
- ファイルヘッダー（プログラム情報、更新履歴）
- `#include`ディレクティブ
- マクロ定義・定数定義
- 構造体定義
- 関数プロトタイプ宣言
- 関数実装（`main`関数、共通関数、サブ関数）

### 4. プログラム指示書生成

既存のC言語ソースコードから、プログラム指示書を自動生成します。

**入力**: C言語ソースコード（`.pc`, `.c`, `.h`などのファイル）  
**出力**: プログラム指示書（Markdown形式）

**生成内容**:
- PGM概要説明書（概要）
- プログラム構造図
- 変数定義
- サブプログラム説明書（概要、処理仕様、エラー一覧）
- 詳細処理説明書
- DB_CALL処理

## 使い方

### テストケース生成

プログラム指示書から単体テストケースを生成します。

```
/c-testcase インプット/{PGM_ID}_プログラム指示書.md
```

**実行フロー**:
1. プログラム指示書の読み込み・分析
2. テスト観点の抽出（正常系・異常系・境界値）
3. Markdown形式でテストケース案を提示
4. ユーザーレビュー・承認
5. TSV形式に変換して `アウトプット/{PGM_ID}_テストケース.tsv` として出力

**詳細**: [`.bob/skills/c-testcase/SKILL.md`](.bob/skills/c-testcase/SKILL.md)

### テストデータ生成

承認済みテストケースから実行可能なテストデータを生成します。

```
/c-testdata
```

**実行フロー**:
1. 承認済みテストケース（TSV）の読み込み
2. プログラム指示書・ファイル定義書を参照
3. 入力データ（`INxx.tsv`）・期待出力データ（`OUTxx.tsv`）を生成
4. ユーザーレビュー・承認
5. `アウトプット/{PGM_ID}_テストデータ/` 配下に出力

**詳細**: [`.bob/skills/c-testdata/SKILL.md`](.bob/skills/c-testdata/SKILL.md)

### ソースコード生成

プログラム指示書からC言語ソースコードを生成します。

```
/c-implement インプット/{PGM_ID}_プログラム指示書.md
```

**実行フロー**:
1. プログラム指示書の読み込み
2. プログラムIDの抽出
3. 変数定義部の生成（`{PGM_ID}.h`）
4. 関数実装部の生成（`{PGM_ID}.c`）
5. ユーザーレビュー・承認
6. `アウトプット/{PGM_ID}.h` / `アウトプット/{PGM_ID}.c` として出力

**詳細**: [`.bob/skills/c-implement/SKILL.md`](.bob/skills/c-implement/SKILL.md)

### プログラム指示書生成

C言語ソースコードからプログラム指示書を生成します。

```
/c-instruction インプット/{PGM_ID}.c
```

**実行フロー**:
1. C言語ソースコードの読み込み
2. プログラムIDの抽出
3. 各テンプレートの適用（PGM概要・構造図・変数定義・サブプログラム説明書・詳細処理・DB_CALL）
4. ユーザーレビュー・承認
5. `アウトプット/{PGM_ID}_プログラム指示書.md` として出力

**詳細**: [`.bob/skills/c-instruction/SKILL.md`](.bob/skills/c-instruction/SKILL.md)

## スキル

本プロジェクトでは、AI開発を効率化するための専用スキルを提供しています。

### 主要スキル

| スキル | コマンド | 説明 |
|:-------|:---------|:-----|
| `c-testcase` | `/c-testcase` | プログラム指示書から単体テストケース（TSV）を生成 |
| `c-testdata` | `/c-testdata` | 承認済みテストケースから実行可能なテストデータ（TSV）を生成 |
| `c-implement` | `/c-implement` | プログラム指示書から変数定義部（`.h`）と関数実装部（`.c`）を生成 |
| `c-instruction` | `/c-instruction` | C言語ソースコードからプログラム指示書を生成 |

### 補助スキル

| スキル | 説明 |
|:-------|:-----|
| `assign-testcase-shot` | テストケースにショット番号（ST001, ST002, ...）を割り当て |
| `excel-to-md` | ExcelファイルをMarkdown形式に変換 |
| `xlsx-to-images` | ExcelファイルをシートごとにPNG画像へ変換（大容量ファイルのトークン上限回避用） |
| `reflection` | セッション記憶情報の整理・整合化 |

## 専用モード

特定の工程に集中して作業する際は、専用モードを使用してください。

| モード | ID | 用途 |
|:-------|:---|:-----|
| C Test Case Designer | `c-testcase-designer` | テストケース設計専用 |
| C Test Data Designer | `c-testdata-designer` | テストデータ設計専用 |
| C Source Generator | `c-implementer` | ソースコード実装専用 |

各モードの詳細ルールは [`.bob/rules-*/AGENTS.md`](.bob/) を参照してください。

## セッション記憶システム

本フレームワークは、AIエージェントがセッションを跨いで学習・改善するための記憶システムを搭載しています。

### 記憶システムの構造

記憶システムは3層構造で構成されています：

1. **ログ層** ([`MEMORY/sessions/`](MEMORY/sessions/)): セッション内の活動を時系列で記録
2. **リフレクション層** ([`MEMORY/reflections/`](MEMORY/reflections/)): セッションから重要な学びを抽出・整理
3. **記憶層** ([`MEMORY/knowledge/`](MEMORY/knowledge/)): 次回セッションで参照する知識を保持

### 使用方法

**セッション開始時**:
- `MEMORY/knowledge/persistent.md` を読み込み（必須）
- `MEMORY/knowledge/preferences.md` を読み込み（必須）
- `MEMORY/reflections/` 配下の最新3件を読み込み（推奨）

**セッション中**:
- 重要なやり取り、エラー、ユーザーフィードバックを記録

**セッション終了時**:
- セッションログを完成させる
- リフレクションを生成（エラーまたは学びがある場合）
- 知識ベースを更新

詳細は[`MEMORY/README.md`](MEMORY/README.md)および[`MEMORY/USAGE_GUIDE.md`](MEMORY/USAGE_GUIDE.md)を参照してください。

## 技術仕様

### ファイル命名規則

| 成果物 | 命名規則 | 例 |
|:-------|:---------|:---|
| テストケース | `{PGM_ID}_テストケース.tsv` | `PG5G117_テストケース.tsv` |
| 入力テストデータ | `INxx.tsv` | `IN01.tsv` |
| 出力テストデータ | `OUTxx.tsv` | `OUT01.tsv` |
| ヘッダファイル（変数定義部） | `{PGM_ID}.h` | `PG5G117.h` |
| C言語ソースコード（関数実装部） | `{PGM_ID}.c` | `PG5G117.c` |
| プログラム指示書 | `{PGM_ID}_プログラム指示書.md` | `PG5G117_プログラム指示書.md` |

すべての成果物は `アウトプット/` ディレクトリ配下に出力します（詳細: [`.bob/rules/output-standards.md`](.bob/rules/output-standards.md)）。
